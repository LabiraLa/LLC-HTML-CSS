What are HTML & CSS?

Let's start with HTML

The content layer

HTML (HyperText Markup Language) is what webpages are made out of.

HTML is used to describe each part of a webpage to the browser. To display content properly in the browser, we use tags. Here are a few examples of common HTML tags:

  • <h1> and </h1>
  • <p> and </p>
  • <strong> and </strong>

For the browser to recogize these tags as HTML, we need to create an HTML file.

Good file naming habits

To create an HTML web page, just name your file using the .html extension (much like .pdf for PDF files or .doc for Word files).

  • Use lowercase letters.
    about.html instead of About.html or ABOUT.html
  • Avoid s p a c e s. Use underscores (_) or dashes (-). Dashes are generally preferred for SEO (search engine optimization).
    business_hours.html or business-hours.html instead of business hours.html
    (The browser will interpret it as business%20hours.html)
  • Avoid symbols like # & * in your file name.
  • Keep your file names concise, yet meaningful.
    about.html, contact.html instead of page1.html, page2.html

These file naming tips can be applied to any type of files such as images, folders, CSS, etc.

The Nuts and Bolts of HTML

When writing HTML, we create HTML elements by wrapping our content in tags. These tags describe the content that is inside of them, NOT what they look like.

For example, the paragraph tag (<p>paragraph content</p>) is used to describe the content in between as being a paragraph, not it's color or any other style.

We use different kinds of tags to create multiple elements and all together they make up our HTML Document.

It is important to understand that HTML has very little to do with the style of your website or the look of the content - that is what CSS is for.

Tags that you should no longer use!

It‚Äôs very important to create semantic, clean and valid markup before doing anything else. Older versions of HTML used to include tags that were used for defining styles, which is now handled with CSS instead.

Here are a few examples of old tags that are no longer used:

  • <u>I'm underlined!</u>
  • <marquee>Anyone remember the marquee tag?</marquee>
  • <font size="large">HUGE FONTS!</font>
  • <center>I'm centered text!</center>
  • <b>I'm Bold text!</b>*
  • <i>I'm Italic!</i>*

Different versions of HTML often have different rules. In HTML5, the <b> and <i> tags were re-introduced but with a slightly different meaning. More info here.

Speaking of CSS...

It makes the web pretty.

What is CSS?

CSS (Cascading Style Sheets) is the presentation layer. In other words, it makes websites look nice.

HTML tells the browser what different parts of the page mean, CSS tells the browser what those parts should look like.

For example, p means paragraph in HTML. But if we wanted to make all of our paragraphs red and underlined, in CSS we'd do something like this:

p {
  color: red;
  text-decoration: underline;
}

Going Deeper with HTML

Writing HTML markup

(You can remove the CSS we added between <style> and </style> for now but keep the tags there!)

HTML tags

The structure of a web page is defined by HTML tags. Left & right angled brackets (shift + < and shift + > ) surround the tag name like this:

<tag>

Think about HTML tags like the beams of a house. It's what holds up a web page creates structure.

HTML tags

Tags typically come in pairs — an opening tag and a closing tag with a forward slash:

<tag></tag>

Some examples that you've already seen:

<!DOCTYPE html>
<html>
  <head>
    <title> </title>
  </head>
  <body>
  </body>
</html>

HTML tags: nesting

Tags can also be nested within other tags.

<tag><tag></tag></tag>

Here are the HTML tags that are required for a basic web page. Let's take a closer look at how they are nested.

<!DOCTYPE html>
<html>
    <head>
        <title> </title>
    </head>
    <body>
    </body>
</html>
<!DOCTYPE html>
<html>
    <head>
        <title></title>
    </head>
    <body>
    </body>
</html>

HTML tags: nesting

Nested elements make up the document tree & the relationships are just like a family tree; branches of ancestors, parents, children, siblings, etc.

<!DOCTYPE html>
<html>
    <head><!-- child of html tag, parent of title tag -->
        <title></title>
    </head>
    <body><!-- child of html tag -->
    </body>
</html>
  • <html> is always first and known as the root node. It has no parent nodes.
    All other HTML tags are nested within <html>.
  • <head> is the child of <html> and the parent node of the <title> tag.
  • What is the relationship of <title> to the <head> tag?
  • What is the relationship between the <head> and <body> tags?

Whitespace through indentation (tabs) make your HTML easier to read. Note how the nested elements are indented to make it easier to see the parent/child relationship.

What goes in the <head> and what goes in the <body>?

The code you put in between the <body> of your HTML document will be displayed on your webpage, the part that your users will see. The code included in the <head> of your HTML document will not be displayed in the browser. The information there is only for the browser to "see."

You can think of the <head> as a place to write secret notes to your browser. For example, our styles (CSS) go in the <head>. We don't want the actual CSS markup to show up on our page, but we want the browser to apply the styles.

HTML content: adding text

Heading tags are used for headings/page titles.

There are six levels ranging from <h1> to <h6>. They follow a hierarchy with <h1> being the most significant heading on the page.

<h1>Name of blog</h1>
<h2>Title of blog post</h2>
<h3>Sub headings within blog post</h3>

Paragraphs are represented by the <p> tag.

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

Links are represented by the <a> (anchor) tag. The href attribute is needed to specify the link location (more on this later).

Visit <a href="http://ladieslearningcode.com">Ladies Learning Code</a>.

More about attributes

An attribute is a modifier of an HTML element and are used to render special content such as links, images, or videos because they provide additional information or instruction.

HTML attributes generally appear as name-value pairs, separated by "=". There are different attributes for different HTML elements. They are written within the opening tag of an element, after the element's name:

<tag attribute="value">content to be modified by the tag</tag>

Here's an example of an attribute that we've seen today:

<a href="link.com">Click here</a>

In the above example, a is the tag, href is the attribute and http://link.com is the value.

CSS basics

The presentation layer

CSS & HTML

CSS is closely paired with HTML and doesn't do anything on its own. It's the presentation layer on top of a content base.

If HTML is the beams of the house, CSS is the layout, paint, and decorations.

CSS Syntax

CSS is made up of rules that refer to HTML elements (like <p> and <ul>), and define what those elements should look like.

A CSS rule is made up of a selector and one or more declarations.

A selector tells the browser which element(s) to style. It looks like this:

h2 {}

The above selector will select all <h2> elements and apply our styles to them. The styles (or declarations) will go between the curly brackets.

The curly braces on your keyboard are located beside the "p" key. Shift + { and Shift + }

CSS Syntax

A declaration tells the browser what style to apply to the selected element.

A declaration is always made up of two parts: a property and a value.

A property tells the browser what we are modifying (ex. font, colour, size).

A value tells the browser how to modify the property (ex. Arial, green, small).

A declaration looks like this:

background-color: green;

CSS Syntax

background-color: green;

The value (the part after the colon) is specific to the property type.

Ex. Size-related properties like height, width, or font-size are usually specified in px (pixels), while colour-related properties are specified with a colour name or code.

It's very important to write the syntax correctly, with a colon after the property name and a semi-colon after the value.

Recap: CSS terminology

Selectors: used to select which element(s) to apply the styles to.

Declaration blocks: consists of a list of declarations (rules to follow) wrapped in curly braces { }.

Declarations are written using CSS property:value pairs.

selector { property: value; }
h1 { color: blue; }

Adding CSS to HTML

You write CSS rules in a set of <style></style> tags included in the <head> section of your HTML document like this:

<html>
  <head>
    <style>
      h2 {
        background-color: green;
      }
    </style>
  </head>
  <body>
    
  </body>
</html>

CSS Syntax

When we put selectors and declarations together, they make a CSS rule, which looks like this: 

h2 {
  background-color: green;
}

This rule will select all <h2> elements and change their background colour to green.

Try copying and pasting the above code in between the <style> tags and refreshing the page in your browser.

CSS Syntax

We can add multiple declarations to one selector to modify multiple properties:

h2 {
  background-color: green;
  font-size: 28px;
}

Make sure you always end each declaration with a semi-colon.

Try adding another declaration, using this example. Save & refresh your browser.

In the previous example, we added all our CSS properties to the body selector. Therefore, all of the children and descendants inherited the styles. However, if a more specific selector was used, it would override the inherited values.

}
h1 {
    color: black; /* only the h1 text will be black */
}

Being more specific: using a class

Because we used img as the selector, the border style now applies to all images. We don't want that, so we need a way to target only the profile image.

HTML gives us a way to attach extra identifying information to HTML elements so that we can target them. This is called a class.

Classes are attributes than can be added to any tag and the value is define by you.

<p class="classname"></p>

When choosing a class name, you can follow the same best practices for choosing file names that was discussed earlier.

Targeting an Element with a Class

Selecting a class in CSS requires a leading period (.) followed by the class name with no spaces.

.classname {}

This will only apply the styles to the item with the class:

.classname {
  color: red;
}

The Box Model

This is the trickiest part of CSS, don't worry if it seems confusing at first!

The CSS Box Model describes the way CSS styles the size and spacing of HTML elements.

To understand the box model, we need to understand how the browser "sees" HTML elements.

To the browser, every HTML element is a rectangular box.

If we show the outlines of all the elements in an HTML document, it looks a bit like the next slide:

The Box Model

CSS uses 5 properties to determine the size and spacing of these boxes:

  • margin, padding, border, width, and height
  • These 5 properties make up the Box Model

The content is the text or image visible to the user. The border is... the border. padding adds space inside of the box and margin adds space outside of the box.

Let's look at each property in more detail.

Margin and Padding

There are two CSS properties for adding space around our elements: margin and padding. Let's take a look at an example.

First, adding a border and background color to an element will make it easier to see how these two properties add space to the element.

Let's say we have 3 elements in our HTML document: a <p>, an <h2>, and another <p>. Adding a solid red border and pink background to the <h2> will look like this:

h2 {
  border: 1px solid red;
  background: pink;
}

So far, this looks pretty awful. All the elements are too close to each other.

Margins

The margin property adds space around the element, outside of the border:

h2 {
  margin: 40px;
}

The example adds 40px of space on every side of the element.


Padding

To add some space inside the border, use the padding property: 

h2 {
  margin: 40px;
  padding: 40px;
}

This example adds 40px of space to every side of the element, but it does so within the border.


Note that the background colour extends when using padding but not margin.

Margin vs. Padding

Adding padding to an element makes the element bigger, whereas adding a margin adds space around the element.

If you make a 100px wide element, and then add 40px of padding to it, it will become 180px wide. (100px width + 40px padding left + 40px padding right)

If you add a 40px margin to a 100px wide element, it stays 100px wide.

If you add a 1px border to a 100px wide element, it will become 102px wide. (100px width + 1px border left + 1px border right)

If your element doesn't have a border or a background, you may not be able to see the difference between margin and padding.

When debugging, it's useful to put a background colour on the elements to see where it begins and ends.

More on Margins and Padding

You can also target a specific side of an element by using these properties: margin-top, margin-right, margin-bottom, and margin-left.

h2 {
  margin-top: 40px;
  margin-bottom: 40px;
}

The same goes for padding and border:

h2 {
  padding-bottom: 40px;
  border-left: 1px solid red;
}

Using just padding, border and margin are actually shorthand properties. These properties, as well as many others, can be defined using shorthand. More info here.

Centering the Logo

To centre elements in CSS, we set the margins to auto, which causes the browser to automatically calculate how much margin to add to each side to centre the element.

Instead of specifying both the margin-left and margin-right properties, we can use a shorthand that looks like this:


img {
  margin: 0 auto;
}

The 0 sets both the top margin and the bottom margin, and the auto sets both the left and right margins.

Then target the class selector in your CSS and apply the margin property:

.logo {
  margin: 0 auto;
}

Nothing Happened!

Let's take a few steps back.

Remember the boxes with the red outlines?

Did you notice that the red outlines around the images don't stretch across the whole screen like the other boxes do?

Inline vs Block

This is because images are inline elements. Inline elements do not take up the full width of the page, they only take up as much space as the content requires.

Most elements, like <h2>, <p>, or <ul> take up the whole width of the page, these are called block elements.

Because our logo element is only as wide as the image, we can't center it relative to the page. Luckily, this is easy to fix: we just tell the <img> tag to behave like a block element instead, by setting the display property.

.logo {
  margin: 0 auto;
  display: block;
}

Other Inline Elements

Other common inline elements include <a>, <strong>, and <em>.

Inline elements don't create line breaks, so they are usually tags that you might find inside a paragraph or in the middle of a sentence.

More info on block & inline level elements here.

Layout: Section by Section

Our site looks much better now, but now it's time to style the individual sections.

We'll need a way to target each section independently, grouping elements within the same section together, creating a layout.

Conveniently enough, HTML provides the <section> tag. We'll wrap each section within a <section> tag and add a class to each one, so we can style each section using CSS.

section is specific to the latest version of HTML which is HTML5 and is not supported in older browsers such as Internet Explorer 8 and below.

CSS Child Selectors

Let's adjust some of the text in our banner section and make the paragraph text bigger and reduce the bottom margin of the <h1>.

We need a way to select the paragraph tags in the banner section and only in the banner section. We could add a class to it like we did with the images, but there's a better way.

Remember the parent-child relationship of HTML elements? We can use that relationship to write CSS using the existing class added to the <section> element.

CSS Child Selectors

CSS child selectors are used to select only those tags which are children of another tag.

To use a child selector, add a space between the parent and the child selector.

section img {}

The above selector will select only those <img> tags which are nested within <section> tags.

CSS Child Selectors

Targeting the <p> tag within the banner <section> should look like this:

.banner p {}

Try increasing the font-size of the selected <p> tag.

Use a child selector to select the <h1> tag in the banner section and capitalize the text using text-transform: uppercase.

Your CSS should look something like this:

.banner p {
  font-size: 18px;
}
.banner h1 {
  text-transform: uppercase;

CSS Floats

We want to align the profile image to the left, and allow the text to wrap around it. We do this using CSS Floats.

Remember how all HTML elements are rectangular boxes stacked on top of each other?

We have to break that stacking flow by floating the image to the side. This will cause the text after the image to move up beside it, because we've disrupted the flow of the rectangular boxes.

CSS Floats

Using the existing selector for our profile <img>, add the declaration float: left; to it:

.profile-image {
  float: left;
}

This has 'floated' our image to the left, but there's a problem...can you figure out what it is?

CSS Floats

We broke the flow of rectangular boxes by adding float: left; to our image.

We need to restore the regular stacking of boxes after this section.

We do this by adding overflow: hidden; to the parent element, <section class="about">.

overflow: hidden; forces the element to 'contain' any floated children, so that they don't disrupt the elements after them.

.about {
  overflow: hidden;
}

Styling Multiple Sections at Once

The About, Favourites and Contact sections could use some spacing. Let's add some padding to it.

Since we're adding the same amount of padding to each section, we can create a new class rather than repeating the same declaration in each of these section's CSS. That way, if we want to change or add some CSS for these sections, we can update just one class, instead of three.

Add a class of box to each of the content <section>s (not including the banner section).

<section class="about box"></section>
<section class="favourites box"></section>
<section class="contact box"></section>

When using more than one class, be sure to separate the class names with a space.

Styling Multiple Sections

Let's write a selector for our new class:

.box {}

Add some padding to all our sections:

.box {
  padding: 50px 100px;
}

Reminder: padding is a shortcut for targeting all sides of an element and can take up to 4 values:

padding: all-sides;
padding: top-bottom left-right;
padding: top left-right bottom;
padding: top right bottom left;

Styling Multiple Sections, Continued

We can also use our box class to target all the headings in each of these sections.

Since we're targeting both our <h2> and <h3> tags, we'll write a selector for both, separating them with a comma. This will allow us to use the same declaration block for multiple selectors.

.box h2, .box h3 {}

Change the colour of our headings.

Adding a Menu

Even though this is a one page website, we can still add a menu to the top of our document so our users can navigate the page without scrolling up and down.

Menus are commonly marked up as lists with links inside each list item.

There are two types of lists in HTML: ordered and unordered (numbered and bulleted). Since we don't need numbers, we'll use an unordered list.

Lists require two tags. <ul></ul> indicates where the list starts and ends. For each bullet or list item, use <li></li> which is nested within the <ul> tag. All content must be contained between the <li></li>.

<ul>
  <li>One list item</li>
  <li>Another list item</li>
</ul>

Ordered lists have the same syntax except <ol></ol> is used instead of <ul></ul>.

Adding a Menu, Continued

Add a <ul> with a class of menu to the top of your HTML document, just inside the <body>.

For each list item, add the section headings. Also include a link in each list item. Note the values of the link (more on that later).

Don't forget the closing tag for your unordered list!

<ul class="menu">
  <li><a href="#about">About</a></li>
  <li><a href="#favourites">Favourites</a></li>
  <li><a href="#contact">Contact</a></li>
</ul>

Adding a Menu, Continued

Because lists have distinct default styles, we'll have to apply many different CSS properties to get it to look more like a navigation bar.

Below are some basic styles to get us started.

.menu {
  list-style: none; /* removes bullets */
  padding: 0;
  margin: 0;
  text-align: center;
  border-bottom: 1px solid #434A5C;
  background: #ffffff;
}

Styling the Menu

We need to get our list items to line up in a row.

Remember the display: block; and display: inline; declarations?

Neither of those is quite what we want, because block will create a new line for each element, and inline won't render padding in the list items.

There's a third option that gives us the best of both worlds: display: inline-block;

Write a selector for the list items, and apply display: inline-block; as well as some padding.


.menu li {
  display: inline-block;
  padding: 20px 35px;
}

Styling the Menu Links

Remove the default underline with text-decoration: none;

.menu a {
  text-decoration: none;
  color: #333333;
  font-weight: bold;
  text-transform: uppercase;
  font-family: 'Oswald', sans-serif;
}

Making the menu "sticky"

We can get our menu to stay at the top of the screen while the user scrolls with the position: fixed; declaration.

The position property requires additional properties to align the element (top, bottom, left and right). All four properties don't have to be used at once, just the direction you need to position the element.

The position property also takes the element out of the natural flow so the rectangular box around the menu is no longer a block element (it won't naturally span the width of the page anymore), so we have to set the width to 100%.

.menu {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
}

More about positioning: A List Apart: CSS Positioning 101

Getting the links to work

Remember these href values?

<ul class="menu">
  <li><a href="#about">About</a></li>
  <li><a href="#favourites">Favourites</a></li>
  <li><a href="#contact">Contact</a></li>
</ul>

Since we are creating a link to specific sections on the same page rather than an external page, value for the href is a bit different.

The links in our menu need to match the sections on our page. We can do this by adding an id attribute to each section that matches the link href (minus the # part):

<section id="about" class="about box"></section>
<section id="favourites" class="favourites box"></section>
<section id="contact" class="contact box"></section>

An id value must be unique and can only be used once per page.
